# Documentação e testes
#### O que faz:

- Produz manuais e fluxogramas para o projeto.

- Documenta o funcionamento do sistema e APIs.

- Gera casos de testes para validar funcionalidades.

- Mantém registro de bugs e correções.

#### Como fazer:

- Atualizar o Notion ou docs do projeto.

- Criar testes unitários e integrados (pytest para Python, testes de front-end).

- Registrar resultados e bugs em ferramenta de gestão.

- Revisar documentação periodicamente.

# Gestão do projeto e integração
#### O que faz:

- Coordena as tarefas da equipe (definição, divisão, prazos).

- Gerencia versionamento do código (Git, branches).

- Garante integração das partes (front, back, Arduino).

- Realiza reuniões para alinhamento e feedback.

#### Como fazer:

- Usar Notion/Trello para controle de tarefas e progresso.

- Definir padrões de commits e revisões no Git.

- Criar ambiente de desenvolvimento comum (Docker, por exemplo).

- Fazer reuniões semanais curtas para alinhamento.
